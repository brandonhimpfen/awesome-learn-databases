
# Repository Standard

This document defines the standard for Awesome Learn repositories.

## Naming Convention

Use the format:

awesome-learn-<topic>

Examples:

awesome-learn-python
awesome-learn-sql
awesome-learn-machine-learning

## Structure

Each repository should include:

- README.md
- CONTRIBUTING.md
- LICENSE

Optional:

- scripts/
- data/

## Content Expectations

Repositories should:

- Provide a clear learning path
- Avoid becoming large, unstructured lists
- Focus on progression and understanding
- Maintain consistent formatting

## Quality Guidelines

- Prefer fewer, higher-quality resources
- Ensure all links are relevant and working
- Keep explanations concise and useful

## Maintenance

- Review links periodically
- Remove outdated or broken resources
- Keep the structure intact as the repo grows
